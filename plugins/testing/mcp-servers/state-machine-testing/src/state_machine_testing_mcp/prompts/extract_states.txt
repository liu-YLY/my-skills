你是状态机建模专家。从以下需求文本中抽取所有状态。

需求文本：
{requirement}

抽取规则：
1. 只抽取名词状态（如"待支付"/"已支付"），禁用动词状态（如"支付中"）
2. 每个状态填写 meaning（含义）
3. 标记 is_initial（初始态）和 is_terminal（终态）
4. 终态必须是业务最终态，不是受理态（如"退款申请成功"不是终态，"退款成功"才是）

输出 JSON 数组：
[
  {
    "name": "状态名",
    "meaning": "含义",
    "is_initial": false,
    "is_terminal": false,
    "entry_events": ["进入事件"],
    "invariants": ["不变量1", "不变量2"]
  }
]

若需求未说明的权限/异常路径，不要脑补，留空数组即可。
