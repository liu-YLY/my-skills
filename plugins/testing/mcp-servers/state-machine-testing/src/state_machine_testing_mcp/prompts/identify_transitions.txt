你是状态机建模专家。基于已抽取的状态，识别所有转换。

需求文本：
{requirement}

已抽取状态：
{states}

识别规则：
1. 每条转换：from → to，事件（动词），守卫（前置条件），副作用（后置动作）
2. 每条转换必须标 evidence_type：
   - 需求明确：PRD 直接描述
   - 合理推理：业务常识推理
   - 待确认：PRD 未说明，需澄清
3. 同时识别禁止转换（forbidden）：
   - 终态吸收：终态 → 任意状态
   - 已知不可逆操作：如"已取消 → 已支付"
4. PRD 未说明的权限/异常路径，标 evidence_type: 待确认，不要脑补

输出 JSON：
{

    {
      "from": "源状态",

      "side_effects": ["副作用1"],
      "evidence_type": "需求明确|合理推理|待确认",
      "source": "PRD §x.x 或推理依据"
    }
  ],
  "forbidden": [
    {
      "from": "源状态",

      "reason": "禁止原因",

    }
  ]
}
