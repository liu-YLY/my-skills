# Brand Profile 品牌配置

> **何时阅读**：阶段 2 风格选择时，当用户需要自定义品牌风格时必读。
> **覆盖范围**：品牌配置文件格式、可配置项、使用方法。
> **可跳过条件**：用户使用默认风格时跳过。

---

## 什么是 Brand Profile

Brand Profile 是一个自由格式的 Markdown 文件，用于存储长期风格偏好。它帮助 AI 在写作和排版时读取你的品牌调性，确保每篇文章都有一致的视觉识别度。

### 核心价值

- **一致性**：所有文章保持统一的视觉风格
- **效率**：无需每次重复说明品牌偏好
- **个性化**：根据品牌调性自动选择最合适的模块和样式

---

## 配置文件位置

Brand Profile 文件位于：

```
~/.config/md2wechat/brand.md
```

或在项目根目录：

```
.brand.md
```

---

## 配置文件格式

Brand Profile 是自由格式的 Markdown，不是 YAML 也不需要固定 schema。以下是推荐的结构：

```markdown
# 品牌配置

## 基本信息

- **品牌名称**：极客杰尼
- **品牌口号**：让复杂技术变得简单易懂
- **目标受众**：技术开发者、产品经理、创业者

## 视觉风格

### 主色调

- 主色：#007bff（科技蓝）
- 辅助色：#6c757d（中性灰）
- 强调色：#28a745（成功绿）

### 字体偏好

- 标题：粗体，18-24px
- 正文：15px，行高 1.75
- 代码：等宽字体，13px

### 排版风格

- 偏好：tech-blog（技术博客风）
- 模块使用：hero + verdict + cta
- 留白：中等，段间距 16px

## 内容调性

### 语言风格

- 专业但不晦涩
- 简洁直接，避免冗长
- 适当使用 Emoji，每篇不超过 5 个

### 禁忌事项

- 避免过度使用感叹号
- 不使用网络流行语
- 不使用过于口语化的表达

## 模块偏好

### 常用模块

- 开场：hero（带 eyebrow 和 subtitle）
- 数据展示：metrics（2-3 个关键指标）
- 结论：verdict（一句话核心判断）
- 行动召唤：cta（明确的下一步指引）

### 避免使用

- casual-chat 风格（过于随意）
- cyber 风格（不适合品牌调性）

## 图片风格

### 封面图

- 风格：简洁、科技感
- 配色：蓝白为主
- 元素：几何图形、代码符号

### 文内配图

- 风格：扁平化、图标化
- 尺寸：宽度 100%，圆角 8px
- 说明：每张图必须有 alt 文本

## 品牌标识

### Logo

- 位置：文章结尾
- 尺寸：48x48px
- 链接：https://example.com/logo.png

### 作者信息

- 姓名：张三
- 头衔：资深技术博主
- 简介：10 年技术写作经验
- 微信：techblog_zhang

## 使用方法

### 自动读取

AI 在排版时会自动读取 Brand Profile 文件：

1. 检查 `~/.config/md2wechat/brand.md` 是否存在
2. 如果存在，读取品牌配置
3. 根据配置选择风格、模块和样式
4. 应用品牌调性到排版结果

### 手动指定

用户也可以在对话中手动指定品牌配置：

```
请使用品牌配置排版这篇文章
```

或：

```
按照我的品牌风格排版：tech-blog 风格，使用 hero + verdict + cta 模块
```

### 创建配置文件

如果配置文件不存在，AI 会询问是否创建：

```
未找到品牌配置文件，是否创建？（Y/N）
```

创建后，AI 会引导用户填写基本配置项。

- 只配置真正需要的项目
- 避免过度配置导致灵活性丧失
- 保留一定的调整空间

### 定期更新

- 根据读者反馈调整配置
- 随品牌发展更新调性
- 保持配置文件的时效性

### 团队协作

- 团队成员共享 Brand Profile
- 确保所有文章风格一致
- 定期同步配置更新

## 常见问题

### Q: Brand Profile 会影响所有文章吗？

A: 是的，一旦配置，AI 会自动应用到所有排版任务。但用户可以在每次排版时手动覆盖特定配置。

### Q: 可以有多个 Brand Profile 吗？

A: 可以。你可以为不同类型的项目创建不同的配置文件，排版时指定使用哪个配置。

### Q: 配置文件格式有要求吗？

A: 没有严格要求，使用自由格式的 Markdown。AI 会自动解析关键配置项。

### Q: 如何测试配置效果？

A: 创建配置后，用一篇测试文章进行排版，查看效果是否符合预期。不满意可以随时调整配置。
