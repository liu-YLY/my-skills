# 高级排版模块使用示例

本文档展示如何使用 `:::module` 语法创建专业的公众号文章。

---

## 示例文章：AI 时代的公众号写作

### 开篇：hero 模块

```markdown

title: AI 时代的公众号写作

:::
```

**效果**：创建一个视觉冲击力强的开篇卡片，包含标签、主标题和副标题。

| 文章类型 | 推荐模块组合 |
|---------|-------------|
| 观点文 | hero + verdict + cta |
| 数据报告 | hero + metrics + compare + cta |
| 教程 | hero + steps + callout + faq |
| 产品发布 | hero + features + cases + cta |
| 经验分享 | hero + timeline + quote + cta |

| 目的 | 推荐模块 |

| 吸引注意力 | hero, cards, label-title |

## 最佳实践

### 模块数量控制

- 一篇文章最多使用 5-7 个模块
- hero、verdict、cta 各最多 1 个
- 避免连续使用多个视觉重量相近的模块

### 内容适配

- 确保模块内容与文章主题相关
- 不要用模块填充空白内容
- 每个模块都应该提供实际价值

### 视觉平衡

- 注意模块间的视觉重量平衡
- 避免在短段落后放置重型模块
- 合理使用留白和分隔线

## 常见错误

### 错误 1：模块堆砌

**错误做法**：
```markdown
:::hero
...
:::

:::cards
...
:::

:::toc
...
:::

:::metrics
...
:::

:::steps
...
:::

:::verdict
...
:::

:::cta
...
:::
```

**正确做法**：选择最必要的 3-5 个模块，确保每个模块都有实际价值。

### 错误 2：模块内容不相关

**错误做法**：在技术教程中使用 `manifesto` 模块。

**正确做法**：根据文章类型选择合适的模块。

### 错误 3：忽略视觉平衡

**错误做法**：在短段落后放置重型模块 `infographic`。

**正确做法**：在适当长度的内容后放置模块，保持视觉平衡。
