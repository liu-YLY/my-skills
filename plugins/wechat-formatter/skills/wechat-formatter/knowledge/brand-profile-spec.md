# Brand Profile 配置规范

> **何时阅读**：阶段 2 风格选择时；用户需要自定义品牌风格时；阶段 5 生成 HTML 需要应用品牌覆盖时。
> **覆盖范围**：brand-profile.md 字段定义、加载机制、优先级规则、完整示例。
> **可跳过条件**：用户使用默认风格且无品牌定制需求时跳过。

---

## 一、Brand Profile 是什么

Brand Profile 是一个**自由格式的 Markdown 文件**，存储长期品牌偏好。AI 在排版时自动读取该文件，把品牌调性（主色调、字体、模块偏好、语言风格）应用到所有文章，保证视觉和内容的一致性。

### 核心价值

- **一致性**：所有文章保持统一的视觉风格，建立品牌识别度
- **效率**：无需每次重复说明品牌偏好，AI 自动读取并应用
- **个性化**：根据品牌调性自动选择最合适的模块组合和样式

### 与风格模板的关系

Brand Profile **不替代**风格模板，而是在风格模板之上做品牌覆盖：

- 风格模板（`templates/{style}.md`）定义**结构规则**（章节顺序、引用块用法、Emoji 数量）
- Brand Profile 定义**品牌变量**（主色、字号、模块组合偏好）

二者协同：先按风格模板排版，再用 Brand Profile 覆盖视觉变量。

## 二、配置文件位置

Brand Profile 支持两个位置，按优先级从低到高：

| 优先级 | 位置 | 作用范围 | 适用场景 |
|--------|------|---------|---------|
| 1（全局） | `~/.config/md2wechat/brand.md` | 当前用户的所有排版任务 | 个人创作者统一品牌 |
| 2（项目） | 项目根目录 `.brand.md` | 当前项目下的所有文章 | 团队/项目级品牌 |

**加载规则**：

1. 排版任务启动时，先检查全局配置是否存在
2. 再检查当前工作目录或输入文件所在目录是否有 `.brand.md`
3. 项目配置覆盖全局配置中相同的字段
4. 两个位置都不存在时，使用 SKILL.md 默认设置（tech-blog 风格、15px 字号、无品牌覆盖）

## 三、可配置字段定义

Brand Profile 是自由格式 Markdown，不强制 YAML schema。AI 通过自然语言解析以下字段。下表列出全部可识别字段：

### 3.1 基本信息字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `品牌名称` | 字符串 | ✅ | 品牌或个人名称，用于 author-card 和 brand-banner | 极客杰尼 |
| `品牌口号` | 字符串 | 可选 | 一句话品牌定位，用于 brand-banner 的 slogan | 让复杂技术变得简单易懂 |
| `目标受众` | 字符串 | 可选 | 文章面向的读者群体 | 技术开发者、产品经理 |

### 3.2 视觉风格字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `主色调` | 颜色值（hex 或名称） | ✅ | 品牌主色，覆盖风格 CSS 中的主色 | `#007bff`（科技蓝） |
| `辅助色` | 颜色值 | 可选 | 用于次级强调，覆盖风格 CSS 辅助色 | `#6c757d`（中性灰） |
| `强调色` | 颜色值 | 可选 | 用于成功/警告状态 | `#28a745`（成功绿） |
| `字号` | 枚举（small/medium/large） | 可选 | 全文字号，默认 medium（15px） | medium |
| `排版风格` | 风格代号 | 可选 | 偏好的默认排版风格 | tech-blog |
| `留白` | 描述 | 可选 | 段间距偏好 | 中等，段间距 16px |

### 3.3 字体偏好字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `标题字体` | 字符串 | 可选 | 标题字体族 | 粗体，18-24px |
| `正文字体` | 字符串 | 可选 | 正文字体族 | 15px，行高 1.75 |
| `代码字体` | 字符串 | 可选 | 代码块字体族 | 等宽字体，13px |

### 3.4 内容调性字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `语言风格` | 描述 | 可选 | 内容调性，影响 AI 写作风格 | 专业但不晦涩 |
| `禁忌事项` | 列表 | 可选 | 避免使用的元素或表达 | 避免过度使用感叹号 |

### 3.5 模块偏好字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `常用模块` | 列表 | 可选 | 默认使用的模块组合 | hero + verdict + cta |
| `避免使用` | 列表 | 可选 | 不建议使用的模块或风格 | casual-chat 风格 |

### 3.6 代码块样式字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `代码块背景` | 颜色值 | 可选 | 代码块背景色，覆盖风格 CSS | `#f5f5f7` |
| `代码块文字色` | 颜色值 | 可选 | 代码块文字颜色 | `#1d1d1f` |
| `代码块字体` | 字符串 | 可选 | 代码块字体族 | SF Mono, Fira Code |

### 3.7 品牌标识字段

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| `Logo` | URL | 可选 | 品牌Logo，用于 brand-banner | https://example.com/logo.png |
| `作者姓名` | 字符串 | 可选 | 默认作者名，用于 author-card | 张三 |
| `作者头衔` | 字符串 | 可选 | 作者头衔 | 资深技术博主 |
| `作者简介` | 字符串 | 可选 | 作者简介 | 10 年技术写作经验 |

## 四、配置加载机制

### 4.1 加载时机

Brand Profile 在 SKILL.md 五阶段工作流的以下时机被加载：

| 阶段 | 是否加载 | 用途 |
|------|---------|------|
| 阶段 1 分析内容 | 否 | Brand Profile 不影响内容分析 |
| 🔴 阶段 2 匹配风格 | 是 | 读取 `排版风格` 字段，作为推荐优先级依据 |
| 阶段 3 执行排版 | 是 | 应用 `语言风格`、`禁忌事项`、`常用模块` 到排版 |
| 🔴 阶段 4 输出校验 | 否 | 校验只检查兼容性，不读 Brand Profile |
| 阶段 5 生成 HTML | 是 | 读取视觉字段（`主色调`、`字号`、`代码块样式`），覆盖风格 CSS |

### 4.2 加载流程

```
阶段 2 启动
  ├─ 检查 ~/.config/md2wechat/brand.md
  │   ├─ 存在 → 读取全局配置
  │   └─ 不存在 → 跳过
  ├─ 检查 {工作目录}/.brand.md
  │   ├─ 存在 → 读取项目配置（覆盖全局同名字段）
  │   └─ 不存在 → 跳过
  └─ 合并配置到内存中的 brand_profile 对象

阶段 3 排版
  ├─ 若 brand_profile.排版风格 存在 → 推荐优先级提升
  ├─ 若 brand_profile.常用模块 存在 → 默认插入这些模块
  └─ 若 brand_profile.禁忌事项 存在 → 排版时主动规避

阶段 5 生成 HTML
  ├─ 加载 styles/{style}.md 中的 CSS
  ├─ 提取 brand_profile 中的颜色/字号字段
  ├─ 用 brand_profile 的值替换 CSS 中的对应属性
  └─ 输出最终 HTML
```

### 4.3 SKILL.md 识别机制

SKILL.md 中 Brand Profile 章节定义了 AI 识别 Brand Profile 的指令。AI 在阶段 2 启动时执行以下检查：

1. 调用文件读取能力检查 `~/.config/md2wechat/brand.md`
2. 调用文件读取能力检查当前工作目录的 `.brand.md`
3. 若两个位置都不存在，跳过 Brand Profile 应用，使用 SKILL.md 默认设置
4. 若存在，解析字段并应用到内存对象，后续阶段引用

## 五、优先级规则

### 5.1 三层优先级

从低到高，三层配置决定最终样式：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1（最低） | 风格 CSS（`styles/{style}.md`） | 风格模板的默认样式 |
| 2（中） | Brand Profile 全局配置 | 用户长期偏好 |
| 3（最高） | 用户当次对话指令 | 用户在某次排版中明确指定的覆盖 |

### 5.2 优先级示例

**场景**：用户全局 Brand Profile 设置 `主色调: #007bff`，本次对话说「用深红色排版」。

**结果**：本次对话指令优先，主色调使用深红色（覆盖 Brand Profile）。

**场景**：Brand Profile 设置 `字号: large`，风格 CSS 默认 `font-size: 15px`。

**结果**：Brand Profile 优先，字号使用 17px（large 对应 17px）。

### 5.3 字段覆盖规则

- **颜色字段**：Brand Profile 的 `主色调` / `辅助色` / `强调色` 覆盖风格 CSS 中所有引用主色的属性
- **字号字段**：Brand Profile 的 `字号` 覆盖风格 CSS 的 `#nice` 字号（small=13px / medium=15px / large=17px）
- **代码块样式**：Brand Profile 的 `代码块背景` / `代码块文字色` 覆盖风格 CSS 的 `#nice pre` 和 `#nice pre code`
- **模块偏好**：Brand Profile 的 `常用模块` 作为默认模块组合，用户对话指令可覆盖
- **语言风格**：Brand Profile 的 `语言风格` 和 `禁忌事项` 影响 AI 排版时的措辞选择，不直接覆盖 CSS

## 六、完整配置示例

### 6.1 技术博客品牌（推荐起点）

```markdown
# 品牌配置

## 基本信息

- **品牌名称**：极客杰尼
- **品牌口号**：让复杂技术变得简单易懂
- **目标受众**：技术开发者、产品经理、创业者

## 视觉风格

### 主色调

- 主色：#007bff（科技蓝）
- 辅助色：#6c757d（中性灰）
- 强调色：#28a745（成功绿）

### 字体偏好

- 标题：粗体，18-24px
- 正文：15px，行高 1.75
- 代码：等宽字体，13px

### 排版风格

- 偏好：tech-blog
- 留白：中等，段间距 16px

## 内容调性

### 语言风格

- 专业但不晦涩
- 简洁直接，避免冗长
- 适当使用 Emoji，每篇不超过 5 个

### 禁忌事项

- 避免过度使用感叹号
- 不使用网络流行语
- 不使用过于口语化的表达

## 模块偏好

### 常用模块

- 开场：hero（带 eyebrow 和 subtitle）
- 数据展示：metrics（2-3 个关键指标）
- 结论：verdict（一句话核心判断）
- 行动召唤：cta（明确的下一步指引）

### 避免使用

- casual-chat 风格（过于随意）
- cyber 风格（不适合品牌调性）

## 代码块样式

- 代码块背景：#f5f5f7
- 代码块文字色：#1d1d1f
- 代码块字体：SF Mono, Fira Code, JetBrains Mono

## 品牌标识

### Logo

- 位置：文章结尾
- 尺寸：48x48px
- 链接：https://example.com/logo.png

### 作者信息

- 姓名：张三
- 头衔：资深技术博主
- 简介：10 年技术写作经验
- 微信：techblog_zhang
```

## 七、使用方法

### 7.1 自动读取

AI 在排版时自动执行加载流程（见第四节），无需用户干预。

### 7.2 手动指定

用户可在对话中明确触发 Brand Profile：

```
请使用品牌配置排版这篇文章
```

或指定具体品牌字段：

```
按我的品牌风格排版：主色 #007bff，使用 hero + verdict + cta 模块
```

### 7.3 创建配置

若配置文件不存在，AI 会询问是否创建：

```
未找到品牌配置文件，是否创建？（Y/N）
```

创建后，AI 引导用户填写第三节中的字段。

### 7.4 临时覆盖

单次排版中可临时覆盖 Brand Profile 字段，不影响配置文件：

```
这次用 large 字号排版，其他按品牌配置
```

本次排版字号使用 large，下次排版仍按 Brand Profile 的 medium。

## 八、校验与最佳实践

### 8.1 配置校验

阶段 2 加载 Brand Profile 后，AI 自动校验：

- 颜色值是否符合 hex 格式（`#` 开头 + 6 位 hex）
- 字号是否为 small / medium / large 之一
- 排版风格是否为 6 种风格代号之一
- 常用模块是否在 9 大类清单内

校验失败的字段会被忽略并提示用户修正。

### 8.2 最佳实践

- **精简配置**：只配置真正需要的字段，避免过度配置导致灵活性丧失
- **定期更新**：根据读者反馈调整配置，随品牌发展更新调性
- **团队协作**：团队成员共享 Brand Profile，确保所有文章风格一致
- **保留调整空间**：不要把所有字段都写死，留出对话级覆盖的余地

## 九、常见问题

### Q1：Brand Profile 会影响所有文章吗？

A：是的，一旦配置，AI 会自动应用到所有排版任务。但用户可在每次排版时通过对话指令临时覆盖特定字段。

### Q2：可以有多个 Brand Profile 吗？

A：可以。全局配置 `~/.config/md2wechat/brand.md` 是默认配置；项目根目录的 `.brand.md` 是项目级配置，会覆盖全局配置中相同的字段。不同项目可用不同的 `.brand.md`。

### Q3：配置文件格式有要求吗？

A：没有严格要求，使用自由格式 Markdown。AI 通过自然语言解析第三节列出的字段。推荐使用示例中的结构化格式以便维护。

### Q4：如何测试配置效果？

A：创建配置后，用一篇测试文章进行排版，查看输出是否符合预期。不满意可随时调整配置文件，下次排版立即生效。

### Q5：Brand Profile 和 styles/ CSS 文件冲突时怎么办？

A：Brand Profile 优先级更高。Brand Profile 中的颜色/字号字段会覆盖 styles/ CSS 中对应的属性。详见第五节优先级规则。
