# 苹果风风格排版规则

> **何时阅读**：用户选择 `apple` 风格时必读。
> **覆盖范围**：极简排版、留白规范、层级节奏、整体结构模板。
> **可跳过条件**：未选择此风格时跳过。

---

## 风格定位

极简、优雅、克制。对标 **Apple 官网**和 **WWDC 技术文档**的设计语言——大量留白、字体层级清晰、无多余装饰，让内容本身成为焦点。

**视觉关键词**：极简 / 留白 / 克制 / 优雅 / 呼吸感

**推荐配色**：深空灰 `#1d1d1f` + 蓝 `#0071e3`（Apple 品牌色系，简约高级）

---

## 排版规则

> **⚠️ 自然度优先原则**
>
> 模板提供的是骨架，不是填空题。排版时应优先保证文章**读起来自然流畅**，其次才是符合模板结构。如果某个元素会让文章显得刻意，宁可省略。记住：**读者不会拿着你的模板检查清单逐项比对**——他们只会判断这篇文章好不好读。

### 1. 标题规范

- 正文标题**从 `##` 起步**（`#` 留给公众号标题栏）
- 标题简洁有力，不加编号、不加 Emoji
- 标题前后各空一行
- 二级标题控制在 8 字以内，三级标题控制在 12 字以内

### 2. 文章头部规范

Apple 风不需要导读框或目标框。开头直接切入主题，用一个简洁有力的段落引出全文：

```markdown
在讨论 XX 之前，我们需要先理解一个核心概念。
```

或直接以问题开场：

```markdown
为什么大多数 XX 方案都不够好？
```

**关键**：开头不超过 3 段话就要进入正文，不要铺垫太多。

### 3. 段落规范

- **每段 2-4 行**，苹果风强调呼吸感，段落比其他风格更短
- 段落间空一行
- 重要观点**单独成段**，不与普通内容混在一起
- 避免大段文字墙，必要时拆成多个短段落

### 4. 引用块规范

引用块用于**核心观点总结**，不加竖线边框，用圆角灰底卡片风格：

```markdown
> 核心观点的表述，一句话说清楚。
```

或强调关键结论：

```markdown
> **关键结论**：用一句话概括最重要的发现。
```

- 每篇文章引用块不超过 3-4 个，精而不多
- 不加 Emoji 标签，保持极简

### 5. 代码块规范

- 代码块使用浅灰底色 + 圆角卡片风格
- 代码块前后各空一行
- 代码块前有简短的说明文字
- 不需要文件路径标注，保持简洁
- 关键行用注释说明，不用 `# ←` 标注

```python
def process(data):
    # 关键：这里决定了处理方式
    result = transform(data)
    return result
```

### 6. 列表规范

- 无序列表用 `•`（默认 `-`），有序列表用数字
- 每个列表项简短，不超过 2 行
- 列表之间可以穿插简短段落，不必连续堆叠

### 7. 图片规范

- 使用 `![描述](url)` 占位
- 图片说明写在图片下方，作为独立段落
- 不加编号标签，保持自然

### 8. Emoji 使用

- **极少量使用**，全篇不超过 2 个
- 只在最关键的结论处使用，如文章末尾的总结
- 标题、正文、列表中都不加 Emoji

### 9. 文末规范

简洁收尾，不搞花哨的互动引导。一个段落总结全文要点即可，或者以一个开放性问题结束。

---

## 整体结构模板

```markdown
## 开篇标题

直接切入主题的段落。2-3 句话把核心问题抛出来。

—

## 第一个问题

正文内容。短段落，多留白。

> 核心观点的总结。

代码或示例。

—

## 第二个问题

继续深入，同样的节奏。

—

## 总结

一两句话收尾。可以是一个思考，也可以是下一步的方向。
```

**结构特点**：
- 没有导读框、目标框等正式结构
- 用 `—` 分隔大段，而不是 `---` 分割线
- 节奏感来自留白和段落长短的变化，而不是编号和标签
- 整体感觉像 Apple 的产品页面——干净、直接、有力
