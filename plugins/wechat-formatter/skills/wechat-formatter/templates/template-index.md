# 模板索引与选择指南

> **何时阅读**：阶段 2 匹配风格时必读。
> **覆盖范围**：六种风格的详细描述、适用场景、视觉特征、对比矩阵、推荐建议。
> **可跳过条件**：用户已明确指定风格时跳过，直接读取对应模板文件。

---

## 六种排版风格

### 1. 技术博客风 (`tech-blog`)

**定位**：专业、干净、有呼吸感。对标**美团技术团队**、**字节跳动技术团队**等一线企业技术博客。

**视觉特征**：
- 文章头部带导读框（字数 + 阅读时间 + 一句话概括）
- 主章节使用中文数字编号（一、二、三...），节奏统一
- 核心观点用 `💡 核心观点` 框突出，每 500-800 字一个
- 主要章节间使用 `---` 分割线，给读者视觉断点
- 代码块前后有说明性文字包裹，关键行末尾标注
- 文末有参考资料（脚注式）+ 互动引导

**适用场景**：
- 编程语言特性解析（如"Python 装饰器深入理解"）
- 技术工具介绍与使用（如"Git 高级技巧"）
- 架构设计与原理讲解
- 技术方案对比与分析

**不适用**：纯经验分享、轻松随笔、无代码片段的纯理论文章。

**详细规则** → [tech-blog.md](tech-blog.md)

---

### 2. 教程指南风 (`tutorial`)

**定位**：清晰、结构化、手把手教学。对标**阮一峰的网络日志**教程风格，让零基础读者也能一步步跟下来。

**视觉特征**：
- 文章头部带目标说明框（🎯 学习目标 + ⏱️ 耗时 + 🎚️ 难度）
- 步骤使用 `### Step N — 名称` 统一格式（em dash 分隔）
- 每个步骤遵循五段结构：目标 / 操作步骤 / 预期结果 / TIP / WARNING
- 四种提示框用 `Emoji + 标签 + ｜ + 说明` 格式区分：TIP / WARNING / NOTE / CHECK
- 每个主要阶段结束有 CHECK 验证框
- 代码块带执行说明 + 预期输出
- 文末有 FAQ + 下一步 + 参考资料

**适用场景**：
- 环境搭建 / 配置教程
- 框架入门指南
- 最佳实践操作步骤
- 工具使用教程

**不适用**：纯理论分析、经验漫谈、无步骤流程的内容。

**详细规则** → [tutorial.md](tutorial.md)

---

### 3. 深度干货风 (`deep-dive`)

**定位**：深入、严谨、信息密度高。对标**美团技术团队深度分析**、**阿里技术**等企业级技术复盘。

**视觉特征**：
- 文章头部带摘要框（📖 摘要 + 关键词）+ 阅读建议框（🕐 跳读指引）
- 开头有纯文本目录（编号列表，不依赖锚点）
- 核心章节用 `## N.` 阿拉伯数字编号
- 每个核心章节结束有 `🔑 核心结论` 框（全文 2-5 个）
- 图表占位带编号和描述性标题（📊 图 N：标题）
- 代码块标注文件路径，关键行带注释
- 表格保留内容但添加 `<!-- ⚠️ -->` 注释提醒替换
- 文末参考资料带简短说明（与本文的关联）

**适用场景**：
- 源码分析（如"Spring 启动流程源码剖析"）
- 性能优化复盘与数据对比
- 技术方案设计文档
- 技术攻关经验总结

**不适用**：简短教程、轻松分享、无深度的介绍文章。

**详细规则** → [deep-dive.md](deep-dive.md)

---

### 4. 轻松聊天风 (`casual-chat`)

**定位**：亲和、真诚、像朋友聊天。对标一线技术博主的随笔风格——**不会太水，但也不端着**。

**视觉特征**：
- 没有导读框等正式结构，开场就是一个自然段落
- 每段 1-3 句话，重要句子单独成段
- 大量留白，用 `—` 作为话题分隔符（比 `---` 更轻）
- 代码块讲解完全口语化，有情绪表达
- 全文 8-15 个 Emoji，分布均匀
- 标题口语化，部分带 Emoji
- 结尾必须有互动引导（评论 / 在看 / 转发）

**适用场景**：
- 技术面试经验分享
- 编程学习心得与教训
- 技术行业观察与思考
- Bug 排查趣事与复盘

**不适用**：严肃的技术文档、需要严谨引用的内容、企业内部报告。

**详细规则** → [casual-chat.md](casual-chat.md)

---

### 5. 苹果风 (`apple`)

**定位**：极简、优雅、克制。对标 **Apple 官网**和 **WWDC 技术文档**的设计语言。

**视觉特征**：
- 没有导读框、目标框等正式结构，直接切入主题
- 每段 2-4 行，强调呼吸感和留白
- 标题不加编号、不加 Emoji，极简风格
- 引用块用圆角灰底卡片，无竖线边框
- 代码块浅灰底色 + 圆角卡片风格
- 全篇 Emoji 不超过 2 个
- 用 `—` 分隔大段，视觉轻盈

**适用场景**：
- 产品设计理念分享
- 架构设计哲学探讨
- 技术选型思考与对比
- 追求高级感的技术文章

**不适用**：需要密集标注的教程、步骤操作指南、大量代码展示的文章。

**详细规则** → [apple.md](apple.md)

---

### 6. 赛博朋克风 (`cyber`)

**定位**：暗黑、酷炫、前沿感。对标**黑客文化**和**极客社区**的视觉语言。

**视觉特征**：
- 深色背景 + 霓虹色调（青色 `#00f0ff` + 紫色 `#ff00ff`）
- 引用块像「系统弹窗」，霓虹紫边框 + 半透明暗色背景
- 代码块极深色背景 + 霓虹绿文字，模拟终端效果
- 标题带技术感，可用少量 Emoji
- 编号章节有「任务推进」的节奏感
- 全篇 5-8 个 Emoji（⚡🔥🚀💻🔑⚠️）
- 分割线有渐变霓虹效果

**适用场景**：
- 安全技术、渗透测试、逆向工程
- 前沿技术探索（AI、区块链、量子计算）
- 极客工具介绍、黑客文化分享
- 追求个性和视觉冲击的技术文章

**不适用**：正式的企业技术文档、面向初学者的教程、严肃的学术分析。

**详细规则** → [cyber.md](cyber.md)

## 快速选择矩阵

| 你的文章特征 | 推荐风格 |
|-------------|---------|
| 大量代码 + 技术讲解 | `tech-blog` |
| 步骤操作 + 读者需要跟着做 | `tutorial` |
| 深度分析 + 图表 + 引用 | `deep-dive` |
| 经验分享 + 观点 + 轻松氛围 | `casual-chat` |
| 追求极简优雅、高级感 | `apple` |
| 暗黑酷炫、前沿极客风 | `cyber` |
| 代码 + 步骤操作都有 | `tutorial`（优先） |
| 观点 + 分析 + 少量代码 | `tech-blog` 或 `casual-chat` |
| 3000+ 字 + 多级结构 | `deep-dive` |

## 选择流程

```
用户文章 → 识别核心内容类型 → 展示 6 种风格简介 → 用户选择 → 加载对应模板
```

**关键规则**：
1. 必须向用户展示六种风格的**一句话简介 + 核心视觉特征**，让用户做出选择
2. 可根据文章内容给出**推荐意见**（如"您的文章代码较多、结构清晰，推荐 `tech-blog` 风格"）
3. 不得替用户做最终决定
4. 用户选择后，**立即加载**对应模板文件的完整规则
