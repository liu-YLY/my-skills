## 通用缺陷模式

### 输入校验类

| 模式 | 测试方法 |
|------|----------|
| 必填绕过 | 前端校验但后端未校验，可通过 API 直接绕过 → 同时测前端 + 直接调 API |
| XSS 注入 | 输入 `<script>` 标签未转义 → 输入框注入 HTML/JS |
| SQL 注入 | 输入拼接到查询中 → 输入 `' OR 1=1 --` |
| 超长截断 | 前端限制长度但后端未限制 → 输入超过限制的文本 |
| 编码问题 | 中文/emoji/特殊字符导致乱码或报错 → 输入多种字符集 |

### 状态与流程类

| 模式 | 测试方法 |
|------|----------|
| 状态回退 | 已到终态的资源被非法操作回退 → 对终态资源执行中间操作 |
| 重复提交 | 快速双击导致重复创建 → 快速连续点击提交按钮 |
| 并发冲突 | 两人同时编辑同一资源 → 双浏览器同时修改并保存 |
| 幽灵状态 | 页面显示状态与后端不一致 → 操作后刷新页面对比 |
| 操作中断 | 操作过程中网络断开或页面刷新 → 提交过程中断网或 F5 |

### 权限与安全类

| 模式 | 测试方法 |
|------|----------|
| 越权访问 | 篡改 URL 中的 ID 可访问他人资源 → 替换 URL 中的资源 ID |
| 水平越权 | 同级别用户可操作他人数据 → A 用户操作 B 用户的数据 |
| 垂直越权 | 低权限用户通过 API 调用高权限接口 → 普通用户直接调管理员 API |
| Token 过期 | 长时间未操作后执行敏感操作 → 等待 token 过期后操作 |
| 敏感数据泄漏 | API 返回了不该返回的字段 → 检查接口返回是否包含密码/密钥 |

### 数据与接口类

| 模式 | 测试方法 |
|------|----------|
| 空数据 | 列表为空时页面异常或报错 → 新账号无数据时访问列表 |
| 分页错误 | 末页/跨页操作后数据丢失或重复 → 删除当前页最后一条后的表现 |
| 排序不稳定 | 相同排序值的数据顺序不固定 → 多次刷新检查顺序一致性 |
| 时区偏移 | 日期时间显示与用户时区不匹配 → 切换浏览器时区后检查 |
| 精度丢失 | 浮点数计算或大整数传递精度丢失 → 涉及金额/ID 时用边界值检查 |

### UI 与交互类

| 模式 | 测试方法 |
|------|----------|
| 按钮状态不同步 | 条件满足后按钮仍置灰 → 逐步填写必填项观察按钮变化 |
| 弹窗残留 | 关闭弹窗后再次打开仍保留上次数据 → 关闭后重新打开检查表单 |
| 加载态缺失 | 操作后无 loading → 在慢网络下操作观察反馈 |
| 错误提示不准确 | 提示信息与实际错误不匹配 → 触发各类错误检查文案 |

## 缺陷模式 → 测试点映射

设计用例时，按以下规则自动从缺陷模式库中补充测试点：

| 功能特征 | 应检查的缺陷模式 |
|----------|------------------|
| 有输入框 | 必填绕过、超长截断、编码问题、XSS（如果面向公网） |
| 有提交按钮 | 重复提交、操作中断 |
| 有列表页 | 空数据、分页错误、排序不稳定 |
| 涉及权限 | 越权访问、水平/垂直越权 |
| 有状态流转 | 状态回退、幽灵状态 |
| 有日期时间 | 时区偏移 |
| 有弹窗/抽屉 | 弹窗残留、关闭行为 |
| 涉及金额/ID | 精度丢失 |

## 领域特定核心模式

### 支付/金额

- **金额精度**：0.1+0.2≠0.3（浮点误差）→ 涉及金额计算时用整数/分测试
- **重复支付**：网络超时重试导致多次扣款 → 同一订单快速提交 2 次
- **部分退款**：退款金额>支付金额 → 退款金额边界：0、等于支付、大于支付

### 搜索/筛选

- **特殊字符**：搜索含 `%` `_` 等通配符报错 → 输入 SQL/正则通配符搜索
- **空结果**：无匹配时页面崩溃 → 搜索不可能存在的内容
- **大结果集**：搜索结果超万条时性能退化 → 搜索高频词，检查分页和性能

### 通知/消息

- **重复通知**：同一事件触发多次通知 → 快速操作触发同一事件
- **通知丢失**：事件发生但通知未送达 → 检查所有通知渠道的投递
- **模板渲染**：变量未替换显示为 `${name}` 原文 → 触发含变量模板的通知

### 文件上传

- **类型绕过**：改扩展名上传非法文件 → .exe 改名为 .png 上传
- **大文件**：超限文件导致 OOM 或超时 → 上传超过限制的文件
- **并发上传**：多文件同时上传部分丢失 → 同时选择 10 个文件上传

## 安全专项检查清单

涉及安全相关功能时，对照此清单补充测试点：

| 检查项 | 测试要点 |
|--------|----------|
| SQL 注入 | 输入框、搜索框、筛选条件中注入 `' OR 1=1 --` |
| XSS 注入 | 输入 `<script>`/`<img onerror>` 等标签 |
| 越权访问 | 篡改 URL 中的资源 ID 访问他人数据 |
| 鉴权缺失 | 未登录/Token 过期直接访问受保护接口 |
| 明文传输 | 检查密码/密钥是否通过 HTTPS 传输 |
| 敏感数据泄漏 | API 响应中是否包含密码/密钥/内部 ID |
| 密码安全 | 弱密码校验、密码复杂度要求、加密存储 |
| 短信/邮箱验证 | 验证码有效期、频率限制、暴力破解防护 |
| CSRF 防护 | 关键操作是否有 Token 校验 |
| 数据健壮性 | 超大 payload、畸形 JSON、非法编码 |
