# 效率度量与效果追踪

> **何时阅读**:阶段 4 输出效率报告时查阅;复盘迭代效率时查阅。
> **覆盖范围**:度量指标定义 / 计算方法 / 输出模板 / 迭代对比。
> **可跳过条件**:用户不要求效率报告时。

## 核心度量指标

| 指标 | 定义 | 计算方法 | 目标值 |
|------|------|----------|--------|
| AI 生成占比 | AI 生成用例数 / 总用例数 | 统计 source:ai-generated 的用例 | >= 70% |
| 工时节省率 | (传统工时 - 实际工时) / 传统工时 | 基于经验估算传统工时 | >= 50% |
| 效率提升倍数 | 传统工时 / 实际工时 | 直接计算 | >= 2x |
| 人工补充率 | 人工补充用例数 / 总用例数 | 统计 source:manual 的用例 | <= 30% |
| 审核修改率 | 审核后修改的 AI 用例数 / AI 生成总数 | 人工跟踪 | <= 20% |

## 传统工时估算基准

根据功能复杂度估算传统模式（纯人工编写）所需工时：

| 复杂度 | 特征 | 传统工时基准 |
|--------|------|-------------|
| 低 | 单表单/单页面，3-5 个字段 | 0.5 人天 |
| 中 | 多表单/多页面，含联动/权限 | 1-2 人天 |
| 高 | 复杂业务流程，多角色多状态 | 3-4 人天 |
| 极高 | 跨系统集成，核心业务链路 | 5+ 人天 |

## 效率报告输出模板

在阶段 4 完成后输出：

```markdown
## 效率度量报告

| 指标 | 本次值 |
|------|--------|
| 用例总数 | X 条 |
| AI 生成 | Y 条 (Z%) |
| 人工补充 | W 条 |
| 预估传统工时 | N 人天 |
| 实际工时 | M 人天 |
| 工时节省 | (N-M)/N * 100% |
| 效率提升 | N/M 倍 |

**质量观察**：
- AI 生成用例中需修改的比例：约 A%
- AI 补充的异常/安全场景数量：B 条
```

## 迭代对比（可选）

当用户要求跨迭代对比时，汇总历史效率数据进行趋势分析：

```markdown
## 迭代效率趋势

| 迭代 | 用例总数 | AI 占比 | 工时节省 | 效率提升 |
|------|----------|---------|----------|----------|
| V1.0 | 80 | 65% | 45% | 1.8x |
| V1.1 | 120 | 74% | 55% | 2.2x |
| V2.0 | 150 | 82% | 63% | 2.7x |

**趋势**：随着 Prompt 策略优化和团队经验积累，AI 占比和效率提升持续增长。
```
