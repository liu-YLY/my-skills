# 阶段 4:自检与补全

> **何时阅读**:进入阶段 4 时必读;快速模式下可只跑「质量检查」+「遗漏扫描」两节。
> **覆盖范围**:Schema 校验确认 / 可追溯性 / 分层与合规 / 探索式 / 覆盖度 / 优先级比例 / 质量 8 原则 / 遗漏扫描 / AI 生成质量检查 / 效率度量输出。
> **行为约束**:发现遗漏直接补充到用例,不另外说明。
> **依赖**:优先级比例参考 [knowledge/test-standards.md](../knowledge/test-standards.md) 三步法;效率度量参考 [knowledge/efficiency-metrics.md](../knowledge/efficiency-metrics.md)。

输出用例后,**必须执行**以下检查(快速模式可压缩)。

## 前置：Schema 校验确认（如有适配器）

若阶段 3 使用了项目适配器，确认适配器指定的校验**已通过**。若未执行或失败，先回到阶段 3 修复。

## 用例自检清单

```markdown
## 用例自检

**可追溯性**：
- [ ] 主要用例与测试点编号可对应；追溯信息（req_ref/trace 或 description 首行）完整

**分层与合规（按需）**：
- [ ] 若本次为接口/契约导向：用例步骤与预期是否与分层一致（非仅 UI 套壳）
- [ ] 若涉及 PII/导出/日志/权限：相关测试点已覆盖

**探索式（若本次输出了探索章程）**：
- [ ] 已记录证据（截图、日志、trace、请求/响应）与结论或待跟进项

**覆盖度检查**：
- [ ] 所有测试点均已有对应用例
- [ ] 正向 Happy Path 全覆盖
- [ ] 每个输入字段的反向校验全覆盖
- [ ] 角色/权限矩阵全覆盖
- [ ] 联动关系全覆盖
- [ ] 边界条件全覆盖
- [ ] 场景法覆盖：基本流 + 备选流 + 异常流

**优先级比例检查**（参照 [knowledge/test-standards.md](knowledge/test-standards.md) 三步法）：
- [ ] P0 占比 10%~15%
- [ ] P1 占比 30%~40%
- [ ] P2 占比 30%~40%
- [ ] P3 占比 10%~15%
- [ ] 如偏离比例，需重新评估升降级
> 若适配器降级了优先级（如 TEST 中 P3→P2），比例以通用格式的原始划分为准。

**质量检查**（对照 8 大设计原则）：
- [ ] 有效性：步骤无歧义，按流程顺序编排
- [ ] 清晰性：包含具体输入数据和预期输出
- [ ] 易操作性：任何人可独立执行，不依赖口头说明
- [ ] expected_results 无模糊描述（禁止「正常」「正确」「符合预期」）
- [ ] steps 具体到可操作（包含实际输入值和操作对象）
- [ ] steps 与 expected_results 数量一致，一一对应（一步一验）
- [ ] ID 无重复

**遗漏扫描**（最后再过一遍 7 个维度）：
- [ ] 是否遗漏了某个字段/按钮/链接？
- [ ] 是否遗漏了某个用户角色？
- [ ] 是否遗漏了取消/返回/刷新等辅助操作？
- [ ] 涉及安全时：是否对照了安全测试检查清单？

**AI 生成质量检查**（仅 AI 生成模式）：
- [ ] AI 生成用例是否覆盖了所有测试点中的标准场景（正常/边界/异常）
- [ ] 人工补充的用例是否覆盖了 AI 容易遗漏的领域特有业务逻辑
- [ ] AI 生成用例的 source 标记是否准确（ai-generated → 审核后 → ai-reviewed）
- [ ] 业务逻辑准确性：AI 是否正确理解了业务规则
```

**如果发现遗漏，直接补充到用例中，不另外说明。**

---

## 效率度量输出（AI 生成模式必做）

阶段 4 自检完成后，输出效率报告（详见 [knowledge/efficiency-metrics.md](../knowledge/efficiency-metrics.md)）：

```markdown
## 效率度量报告

| 指标 | 本次值 |
|------|--------|
| 用例总数 | X 条 |
| AI 生成 | Y 条 (Z%) |
| 人工补充 | W 条 |
| 预估传统工时 | N 人天 |
| 实际工时 | M 人天 |
| 工时节省 | (N-M)/N * 100% |
| 效率提升 | N/M 倍 |

**质量观察**：
- AI 生成用例中需修改的比例：约 A%
- AI 补充的异常/安全场景数量：B 条
```

**统计方法**：根据用例中的 `source` 字段统计 AI 生成 vs 人工补充数量。
