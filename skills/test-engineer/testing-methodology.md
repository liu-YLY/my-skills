# 测试方法论与实践参考（精简）

> **何时阅读**:阶段 2 选择具体设计方法时按需查阅(等价类/边界值/状态迁移/判定表/场景法);Bug 根因分析时查阅 5 Whys。
> **可跳过条件**:阶段 2 已选定方法且熟悉其用法;非 Bug 分析任务。大多数情况只需阶段 2 决策树即可。

## 测试设计技术速查

### 等价类划分 + 边界值分析

输入域划分有效/无效等价类，边界取：边界值、边界值-1、边界值+1。

```
年龄 [18,60]：测试值 17,18,19,59,60,61；无效：<0, "abc", "", 小数
```

### 状态迁移法

识别所有状态 → 识别触发事件 → 绘制状态迁移图 → 导出迁移表 → 设计覆盖路径（0-switch/1-switch）。

### 判定表法

多条件组合场景，列举所有条件组合及对应动作。当某条件不影响动作时可用 "-" 合并简化。

### 场景法

基本流(Happy Path) → 备选流(取消/修改/切换) → 异常流(错误输入/网络断开/超时)。

## 测试策略

| 风险等级 | 测试深度 |
|----------|----------|
| 高（核心业务） | 全面覆盖 + 探索性 |
| 中（常用功能） | 等价类 + 边界值 |
| 低（辅助功能） | 冒烟测试 |

回归策略：全量回归(大版本)、影响域回归(代码变更)、冒烟回归(每次构建)。

## 缺陷分类

| 类型 | 典型表现 |
|------|----------|
| 功能缺陷 | 操作无响应、结果错误 |
| 数据缺陷 | 计算错误、数据丢失 |
| 接口缺陷 | 字段缺失、类型错误 |
| 性能缺陷 | 加载慢、接口超时 |
| 安全缺陷 | 越权、注入攻击 |
| 兼容性缺陷 | 浏览器/系统差异 |

## 根因分析（5 Whys）

从问题表象逐层追问"为什么"，直到找到可落地的根因。示例：

```
用户下单后库存未扣减
→ 扣减接口超时
→ 连接池耗尽
→ 慢查询占满连接
→ 缺少索引
→ 新增字段时未评审 SQL 性能 ← 根因
```
