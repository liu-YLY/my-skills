## 测试设计技术详解

### 等价类划分

将输入域划分为若干等价类，从每个类中选取代表值。

**实践要点：**
- 有效等价类：至少覆盖一次
- 无效等价类：每个单独覆盖（避免屏蔽效应）

```
示例：年龄输入框（范围 18-60）
有效等价类：[18, 60] → 取 30
无效等价类1：< 18 → 取 15
无效等价类2：> 60 → 取 65
无效等价类3：非数字 → 取 "abc"
无效等价类4：空值 → 取 ""
无效等价类5：小数 → 取 25.5
```

### 边界值分析

在等价类边界上下取值，通常取：边界值、边界值-1、边界值+1。

```
示例：年龄 [18, 60]
测试值：17, 18, 19, 59, 60, 61
增强边界：0, -1, MAX_INT
```

### 状态迁移法

适用于有明确状态流转的系统。

**绘制步骤：**
1. 识别所有状态
2. 识别所有触发事件
3. 绘制状态迁移图
4. 导出状态迁移表
5. 设计覆盖路径（0-switch/1-switch）

```
示例：订单状态
[待支付] --支付成功--> [已支付] --发货--> [已发货] --签收--> [已完成]
[待支付] --取消----> [已取消]
[待支付] --超时----> [已关闭]
[已支付] --退款----> [退款中] --退款成功--> [已退款]
```

### 判定表法

适用于多条件组合场景，列举所有条件组合及对应动作。

```
示例：登录校验
条件1：用户名正确  Y Y N N
条件2：密码正确    Y N Y N
────────────────────────────
动作：登录成功     Y
动作：密码错误提示   Y
动作：用户不存在     Y Y
```

**简化规则：** 当某个条件的取值不影响动作时，可用 "-" 合并。

### 场景法

识别基本流和备选流，构造端到端场景。

**结构：**
- 基本流（Happy Path）：最正常的使用路径
- 备选流（Alternative）：用户可能的其他合理操作
- 异常流（Exception）：错误输入、网络断开、超时等

## 测试策略

### 测试金字塔

```
        /  E2E  \        ← 少量，验证关键业务流
       / 集成测试 \       ← 适量，验证模块交互
      /  单元测试  \      ← 大量，验证代码逻辑
```

### 风险驱动测试

根据功能的风险等级分配测试力度：

| 风险等级 | 测试深度 | 关注点 |
|----------|----------|--------|
| 高（核心业务） | 全面覆盖 + 探索性测试 | 正向/反向/边界/并发/安全 |
| 中（常用功能） | 等价类 + 边界值 | 正向/主要反向 |
| 低（辅助功能） | 冒烟测试 | 基本可用性 |

### 回归测试策略

- **全量回归**：大版本发布前
- **影响域回归**：根据代码变更影响范围选择用例
- **冒烟回归**：每次构建后执行核心路径

## 缺陷管理

### Bug 报告模板

```markdown
**标题**：[模块名] 具体问题描述

**环境**：
- 系统/浏览器版本
- 测试环境（staging/production）
- 账号类型/权限

**复现步骤**：
1. 步骤 1
2. 步骤 2
3. 步骤 3

**实际结果**：具体描述观察到的行为
**预期结果**：根据需求/规范应有的行为

**严重程度**：Critical/Major/Minor/Trivial
**优先级**：P0/P1/P2/P3

**附件**：截图/录屏/日志
```

### 缺陷分类

| 类型 | 描述 | 典型表现 |
|------|------|----------|
| 功能缺陷 | 功能不符合需求规格 | 操作无响应、结果错误 |
| 数据缺陷 | 数据处理逻辑错误 | 计算错误、数据丢失 |
| 接口缺陷 | API 契约不一致 | 字段缺失、类型错误 |
| 性能缺陷 | 响应超出可接受范围 | 页面加载慢、接口超时 |
| 安全缺陷 | 存在安全漏洞 | 越权访问、注入攻击 |
| 兼容性缺陷 | 特定环境下异常 | 浏览器差异、系统差异 |

### 根因分析方法

**5 Whys 法：**
```
问题：用户下单后库存未扣减
Why 1：扣减库存接口返回超时
Why 2：数据库连接池耗尽
Why 3：慢查询占满连接
Why 4：缺少索引导致全表扫描
Why 5：新增字段时未评审 SQL 性能 ← 根因
```

**鱼骨图分类维度：**
- 人（操作失误、权限问题）
- 机（服务器、网络、配置）
- 料（数据质量、依赖服务）
- 法（流程缺陷、规范缺失）
- 环（环境差异、版本差异）
