# 微信公众号 Markdown 兼容性规范

> **何时阅读**：阶段 4 校验时必读。
> **覆盖范围**：公众号支持的 Markdown 语法、不兼容列表、替代方案、表格/代码块特殊处理。
> **可跳过条件**：无。排版输出必须通过公众号兼容性检查。

---

## 公众号编辑器支持的 Markdown 语法

| 语法 | 支持情况 | 备注 |
|------|---------|------|
| `## 标题` / `### 标题` | ✅ 支持 | `#` 不推荐（与公众号标题重复） |
| `**粗体**` | ✅ 支持 | |
| `*斜体*` | ⚠️ 弱支持 | 中文字体下斜体效果不明显，建议少用 |
| `` `行内代码` `` | ✅ 支持 | 公众号会显示灰色背景 |
| ` ```代码块``` ` | ⚠️ 部分支持 | 不支持语言高亮，但保留等宽字体格式 |
| `- 列表` / `1. 列表` | ✅ 支持 | |
| `> 引用` | ✅ 支持 | |
| `[文字](链接)` | ❌ 不支持 | 公众号不解析 Markdown 链接 |
| `![图片](链接)` | ❌ 不支持 | 需手动上传图片 |
| `| 表格 |` | ❌ 不支持 | 需转为图片或使用公众号表格功能 |
| `---` 分割线 | ✅ 支持 | |
| `~~删除线~~` | ❌ 不支持 | |
| HTML 标签 | ⚠️ 部分支持 | 仅限安全标签 |

## 不兼容语法及替代方案

### 1. 链接处理

```markdown
❌ Markdown 链接（公众号不解析）：
详见 [Python 官方文档](https://docs.python.org/3/)

✅ 替代方案 A — 脚注式（推荐）：
详见 Python 官方文档 [1]。

---

**参考资料**

[1] Python 官方文档: https://docs.python.org/3/
[2] PEP 8 规范: https://peps.python.org/pep-0008/

✅ 替代方案 B — 直接标注（短链接）：
详见 Python 官方文档（docs.python.org/3）
```

### 2. 代码块处理

```markdown
❌ 期望语法高亮（公众号不支持）：
```python
def hello():
    print("Hello")
```

✅ 实际效果：
公众号会显示为等宽字体灰色背景块，但无语法高亮。

✅ 优化建议：
- 代码块前后加空行
- 复杂代码可截图后插入图片（提示用户）
- 代码块上方用文字说明：「以下是 Python 代码：」
```

### 3. 表格处理

```markdown
❌ Markdown 表格（公众号不支持渲染）：
| 方案 | QPS | 延迟 |
|------|-----|------|
| A    | 1000| 50ms |

✅ 替代方案 A — 列表化（≤3列推荐）：
- **方案 A**：QPS 1000，延迟 50ms
- **方案 B**：QPS 2000，延迟 30ms

✅ 替代方案 B — 图片化（>3列推荐）：
（建议用 Excel 制作表格后截图插入）
<!-- 请将以下表格截图后替换：方案对比表 -->
![方案对比表](https://your-cdn.com/comparison.png)
```

### 4. 图片处理

```markdown
✅ 排版文件中的占位格式：
![装饰器执行流程图](https://your-cdn.com/decorator-flow.png)
<!-- ⚠️ 请在公众号编辑器中手动上传此图片替换占位符 -->

✅ 输出时给用户的提醒：
> 📷 **图片上传提醒**：本文共包含 3 处图片占位，请在发布前逐一替换为
> 实际图片。建议图片宽度 900px，格式 PNG/JPG。
```

### 5. 删除线

```markdown
❌ ~~不推荐的功能~~ （公众号不支持）
✅ **推荐的功能** （用粗体强调替代）
```

## 排版输出文件规范

### 文件命名

```
{原文件名}_formatted_{风格代号}.md

示例：
python-decorator_formatted_tech-blog.md
testing-guide_formatted_tutorial.md
```

### 文件头部信息

排版后的文件应在开头添加元信息注释：

```markdown
<!--
排版风格：技术博客 (tech-blog)
排版时间：2026-05-12
原始文件：python-decorator.md
图片占位：3 处（需手动上传）
代码块：5 处
-->
```

### 图片清单

文件末尾应附图片上传清单：

```markdown
---

## 📷 图片上传清单

发布前请在公众号编辑器中上传以下图片：

1. **装饰器执行流程图** — 建议尺寸 900×600px
2. **性能对比图表** — 建议尺寸 900×500px
3. **架构示意图** — 建议尺寸 900×700px

> 💡 提示：推荐使用「壹伴」或「秀米」等公众号排版工具一键替换图片。
```

## 发布前最终检查

| 检查项 | 操作 |
|--------|------|
| 代码块是否完整 | 将 Markdown 复制到公众号编辑器，检查代码块是否被截断 |
| 引用块格式 | 检查 `>` 引用块是否正常渲染 |
| 链接替换 | 按替代方案处理所有 Markdown 链接 |
| 图片替换 | 逐一上传图片替换占位符 |
| 表格替换 | Markdown 表格需转为图片或公众号表格 |
| 段落间距 | 检查段落间是否有空行 |
| 中英空格 | 复制到编辑器后有时会丢失，需再次检查 |
