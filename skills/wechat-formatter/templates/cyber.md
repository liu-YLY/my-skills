# 赛博朋克风格排版规则

> **何时阅读**：用户选择 `cyber` 风格时必读。
> **覆盖范围**：暗黑排版、霓虹配色、代码块特效、整体结构模板。
> **可跳过条件**：未选择此风格时跳过。

---

## 风格定位

暗黑、酷炫、前沿感。对标**黑客文化**和**极客社区**的视觉语言——深色背景、霓虹色调、代码发光，让读者感受到技术的锋利和未来感。

**视觉关键词**：暗黑 / 霓虹 / 发光 / 未来 / 极客

**推荐配色**：霓虹青 `#00f0ff` + 霓虹紫 `#ff00ff` + 深空黑 `#0a0a0f`

---

## 排版规则

> **⚠️ 自然度优先原则**
>
> 模板提供的是骨架，不是填空题。排版时应优先保证文章**读起来自然流畅**，其次才是符合模板结构。不要为了赛博感而堆砌装饰——酷是结果，不是目的。记住：**读者不会拿着你的模板检查清单逐项比对**——他们只会判断这篇文章好不好读。

### 1. 标题规范

- 正文标题**从 `##` 起步**
- 标题简洁有力，可以带技术感的词汇
- 标题前可以加一个 Emoji，控制在 1-2 个以内
- 允许标题带编号 `## 1.` `## 2.`，也可以不带

### 2. 文章头部规范

赛博风可以用一个简短的「系统提示」风格开场：

```markdown
> ⚡ **系统启动** | 本文将深入 XX 的核心机制，预计阅读 8 分钟。
```

或者直接以一句有冲击力的话开场：

```markdown
99% 的人不知道，XX 背后隐藏着一个惊人的设计。
```

**关键**：开场要有「加载完成，进入正题」的感觉，不要铺垫太久。

### 3. 段落规范

- **每段 3-5 行**，保持信息密度
- 段落间空一行
- 重要结论可以**单独成段**并加粗
- 技术名词、命令行、变量名使用行内代码

### 4. 引用块规范

引用块用于**核心结论**或**系统提示**，霓虹紫边框 + 半透明暗色背景：

```markdown
> **🔑 核心结论**：XX 的本质是 YY，理解了这一点，后面的 ZZ 就迎刃而解。
```

或用于警告/注意：

```markdown
> **⚠️ 警告**：在生产环境中切勿使用此配置。
```

- 每篇文章引用块 3-6 个，可以比其他风格多一些
- Emoji 标签让引用块像「系统消息」一样有赛博感

### 5. 代码块规范

- 代码块使用**极深色背景** + **霓虹绿文字**，模拟终端/黑客屏幕效果
- 代码块有发光边框效果
- 代码块前后有说明性文字
- 关键行用注释标注

```bash
# 深入系统核心
sudo nmap -sV 192.168.1.0/24  # 扫描目标网段
```

- 支持命令行风格的代码块，带 `$` 前缀
- 可以用伪代码或概念性代码块来解释原理

### 6. 列表规范

- 列表项可以是技术要点、特性罗列、步骤流程
- 列表项尽量简短，不超过 2 行
- 有序列表用于步骤，无序列表用于特性罗列

### 7. Emoji 使用

- **适量使用**，全篇 5-8 个
- 推荐 Emoji：⚡ 🔥 🚀 💻 🔑 ⚠️ 🎯 💡
- 分布在标题、引用块标签、关键结论处

### 8. 文末规范

以一个有赛博感的方式收尾：

```markdown
---

**📡 信号传输完毕。** 如果这篇文章帮你理清了 XX 的脉络，点个在看让更多人收到这份信号。
```

---

## 整体结构模板

```markdown
> ⚡ **系统加载** | 关于 XX 你需要知道的一切。

## 1. 前言

简短引入，2-3 段话。

---

## 2. 核心问题

正文内容。

> **🔑 核心结论**：关键观点。

代码示例。

---

## 3. 深入分析

继续展开。

---

## 4. 总结

> **📡 信号传输完毕。** 收尾 + 互动引导。
```

**结构特点**：
- 用 `---` 分割线做章节间隔，视觉上有「关卡切换」的感觉
- 编号章节让文章有「任务推进」的节奏
- 引用块像「系统弹窗」，提示关键信息
- 整体感觉像在看一份技术解密文档——有悬念、有推进、有结论
