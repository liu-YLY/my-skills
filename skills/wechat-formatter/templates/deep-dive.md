# 深度干货风格排版规则

> **何时阅读**：用户选择 `deep-dive` 风格时必读。
> **覆盖范围**：多级目录、图表占位、结论框、引用规范、整体结构模板。
> **可跳过条件**：未选择此风格时跳过。

---

## 风格定位

深入、严谨、结构强。适合 3000 字以上的长篇技术分析，强调逻辑严密性和信息密度。

---

## 排版规则

### 1. 标题规范

- **一级标题** `#`：仅用于文章主标题
- **二级标题** `##`：用于核心章节
- **三级标题** `###`：用于子主题
- **四级标题** `####`：仅在必要时用于更细粒度拆解（尽量不用）
- 文章开头必须有**目录**（使用纯文本编号列表，不使用锚点链接）

### 2. 目录模板

```markdown
## 目录

1. 背景与问题
2. 核心概念
3. 源码分析
   1. 入口方法
   2. 核心流程
4. 架构设计
5. 性能对比
6. 总结与思考
```

> **注意**：公众号不支持 Markdown 锚点链接，目录使用纯文本编号列表，无需添加链接。

### 3. 结论框

重要结论/核心发现使用特殊引用块突出：

```markdown
> **🔑 核心结论**
>
> Spring 的 Bean 生命周期包含 13 个阶段，其中 `postProcessBeforeInitialization` 和
> `postProcessAfterInitialization` 是最常用的扩展点。理解这两个方法就能覆盖 80% 的
> Spring 扩展开发场景。
```

### 4. 图表占位

本风格大量使用图表来辅助说明，排版时用占位符标记：

```markdown
**📊 图1：Spring Bean 生命周期流程**

![Spring Bean 生命周期](https://your-cdn.com/bean-lifecycle.png)
<!-- 建议使用 Mermaid 或 Draw.io 绘制后导出为 PNG -->

> 上图展示了 Bean 从定义到销毁的完整链路。
```

**图表编号规则**：全文统一编号 `图1`、`图2`...，编号后跟描述性标题。

### 5. 代码引用规范

- 代码块必须标注语言和文件路径（注释形式首行）：

```markdown
**核心入口：`AbstractApplicationContext.java#refresh()`**

```java
// AbstractApplicationContext.java
public void refresh() throws BeansException, IllegalStateException {
    synchronized (this.startupShutdownMonitor) {
        prepareRefresh();
        ConfigurableListableBeanFactory beanFactory = obtainFreshBeanFactory();
        // ...
    }
}
```
```

- 不展示完整源码，只展示关键逻辑
- 代码中关键行用注释标注 `// ← 关键点`

### 6. 数据/对比表格

对于公众号不支持的 Markdown 表格，在排版文件中用注释标记提醒：

```markdown
<!-- 以下表格建议在公众号编辑器中用内置表格功能制作，或截图插入 -->
**表1：方案对比**

| 方案 | 响应时间 (P99) | 吞吐量 (QPS) | 复杂度 |
|------|---------------|-------------|--------|
| 方案 A | 120ms | 8500 | 低 |
| 方案 B | 85ms | 12000 | 中 |
| 方案 C | 65ms | 15000 | 高 |
```

> **注意**：公众号不支持 Markdown 表格渲染，排版输出保留表格内容但添加注释提醒用户替换。

### 7. 整体结构模板

```
# [深度分析标题]

> **📖 阅读建议**：本文约 4000 字，重点在第 3-5 节，建议配合源码阅读。

## 目录
...

## 1. 背景与问题
[问题描述 + 为什么值得深入]

## 2. 核心概念
[关键概念定义，为后续分析打基础]

## 3-N. 深度分析（核心章节，可分多节）
[源码/架构/原理剖析]

> **🔑 核心结论**
> [阶段性总结]

## N+1. 性能对比 / 实践数据
[如有测试数据]

## 总结与思考
- **核心要点**：[3-5 条]
- **局限与改进方向**
- **对工作的启示**

## 参考资料
- [1] 引用标题: url — 简短说明
- [2] 引用标题: url — 简短说明
```

## 排版检查清单

- [ ] 文章开头有纯文本目录
- [ ] 至少 1 个 `🔑 核心结论` 框
- [ ] 图表占位符带编号和标题
- [ ] 代码块首行有文件路径注释
- [ ] 有对比表格（如适用）且带注释提醒
- [ ] 有「总结与思考」章节
- [ ] 有「参考资料」章节（脚注式）
- [ ] 关键代码行有 `// ←` 注释
