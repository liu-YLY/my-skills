# 技术博客风格排版规则

> **何时阅读**：用户选择 `tech-blog` 风格时必读。
> **覆盖范围**：标题规范、代码块格式、段落排版、引用样式、整体结构模板。
> **可跳过条件**：未选择此风格时跳过。

---

## 风格定位

专业、干净、理性。让读者专注于技术内容本身，排版不喧宾夺主。

---

## 排版规则

### 1. 标题规范

- **一级标题** `#`：仅用于文章主标题（对应公众号标题），不在正文中使用
- **二级标题** `##`：用于主要章节，简洁有力，如 `## 什么是装饰器`
- **三级标题** `###`：用于小节，如 `### 带参数的装饰器`
- 标题前后各空一行
- 标题不使用 Emoji（保持专业感）

### 2. 代码块格式

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        print("before")
        result = func(*args, **kwargs)
        print("after")
        return result
    return wrapper
```

**规则**：
- 始终标注语言类型（python / java / go / javascript / bash / yaml 等）
- 代码块前后各空一行
- 代码块上方用一句话说明代码用途，如：`以下是一个简单的装饰器实现：`
- 行内代码使用 `` ` `` 包裹，如 `` `@decorator` ``
- 关键代码行可在描述中用 `**粗体**` 指代

### 3. 段落排版

- 每段 2-4 句话，手机阅读不超过 5 行
- 段落之间空一行
- 核心概念首次出现时用 `**粗体**` 强调
- 列表项用 `-`，嵌套缩进 2 空格
- 有序列表用于步骤说明

### 4. 引用样式

```markdown
> **注意**：装饰器在函数定义时就会执行，而非调用时。
```

用于：
- 重要提示/注意事项
- 补充说明
- 引用外部资料

### 5. 图片占位

```markdown
![装饰器执行流程图](https://your-cdn.com/decorator-flow.png)
<!-- 请替换为实际图片链接 -->
```

### 6. 整体结构模板

```
# [文章标题]（用户可保留）

## 前言 / 背景
[1-2 段简述：为什么写这篇文章、解决什么问题]

## 核心内容 - 章节一
[正文 + 代码 + 说明]

## 核心内容 - 章节二
[正文 + 代码 + 说明]

...

## 总结
[3-5 个要点总结，用列表]

## 参考
[1] 参考资料1: url
[2] 参考资料2: url
```

- [ ] 代码块前后各空一行
