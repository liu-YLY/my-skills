# 技术博客风格排版规则

> **何时阅读**：用户选择 `tech-blog` 风格时必读。
> **覆盖范围**：标题规范、代码块格式、段落排版、引用样式、整体结构模板。
> **可跳过条件**：未选择此风格时跳过。

---

## 风格定位

专业、干净、有呼吸感。对标**美团技术团队**、**字节跳动技术团队**等一线企业技术博客。排版服务于内容，让读者沉浸于技术本身。

**视觉关键词**：留白充足 / 层次分明 / 克制配色 / 代码规整

**推荐配色**：深蓝 `#2b5797` + 灰 `#595959`（商务专业感，引导用户使用 mdnice 等工具应用）

---

## 排版规则

> **⚠️ 自然度优先原则**
>
> 模板提供的是骨架，不是填空题。排版时应优先保证文章**读起来自然流畅**，其次才是符合模板结构。如果某个提示框或编号体系会让文章显得生硬，宁可省略。记住：**读者不会拿着你的模板检查清单逐项比对**——他们只会判断这篇文章好不好读。

### 1. 标题规范

- 正文标题**从 `##` 起步**（`#` 留给公众号标题栏）
- 主章节使用 `## 一、章节名` 格式（中文数字 + 顿号），视觉节奏统一
- 子章节使用 `### 1.1 小节名` 格式
- 标题前后各空一行
- 标题不使用 Emoji（保持专业感）
- 标题文字简洁有力，8-15 字为佳

### 2. 文章头部规范

每篇排版输出**必须**在正文开头包含「导读」信息框：

```markdown
> **📖 导读**
>
> 本文约 **XXXX 字**，预计阅读 **X 分钟**。文章将带你了解 [一句话核心内容]。适合 [目标读者群体] 阅读。
```

导读框紧跟文章标题后，用引用块 `>` 实现。关键数字用粗体标出。

### 3. 代码块格式

**代码块前**：用一句话说明代码的用途和上下文，格式为普通段落。

**代码块本身**：
- 始终标注语言类型（python / java / go / javascript / bash / yaml 等）
- 关键行末尾用 `# ← 关键` 标注重点（仅 1-2 处，不宜过多）
- 代码块前后各空一行

**代码块后**：用 1-2 句话总结这段代码的核心要点或输出含义。

```markdown
✅ 完整的代码块呈现模式：

下面是用 pytest 编写的注册接口自动化测试：

```python
def test_register_success():
    phone = "13800138000"
    password = "Test@123456"

    response = register_api(phone, password)  # ← 核心调用

    assert response.status_code == 200
    assert response.json()["code"] == 0
    assert "token" in response.json()["data"]
```

上面这段代码的关键在于：测试数据不依赖外部环境，断言精确到具体字段值而非仅判断成功/失败。
```

### 4. 段落排版

- 每段 2-4 句话，手机屏幕不超过 5 行
- 段落之间空一行
- 核心概念**首次出现**时用 `**粗体**` 强调（同一词汇只强调一次）
- 中文与英文/数字之间**始终**加空格
- 有序列表 `1.` 用于序列说明，无序列表 `-` 用于并列要点
- 列表前后各空一行

### 5. 引用与提示框

技术博客风格使用 **2 种提示框**：

**A. 核心观点框** — 用于强调关键结论或重要洞察：

```markdown
> **💡 核心观点**
>
> 需求不明确的地方不要做假设，列出问题清单向产品确认。这是避免大规模返工的关键一步。
```

**B. 注意提醒框** — 用于补充说明或注意事项：

```markdown
> **注意**：装饰器在函数定义时就会执行，而非调用时。这个特性经常导致意外的行为。
```

**使用规则**：
- 每 500-800 字可用一个核心观点框，全篇 2-4 个
- 注意框仅在确实需要提醒时使用，不过度
- 提示框前后各空一行

### 6. 章节分隔

主要章节之间使用分割线 `---`，前空一行：

```markdown
[上一节最后一段]

---

## 二、下一章节
```

分割线给读者一个视觉断点，暗示内容切换。小节之间不需要分割线，仅用标题间距即可。

### 7. 图片占位

```markdown
![装饰器执行流程图](https://your-cdn.com/decorator-flow.png)
<!-- ⚠️ 请在公众号编辑器中手动上传替换，建议宽度 900px -->
```

图片上方用普通段落说明图表内容和阅读要点。

### 8. 链接处理

微信公众号不支持 Markdown 链接语法。统一使用**脚注式**：

```markdown
文中引用：
详见 Python 官方文档 [1] 中关于装饰器的说明。

文末集中列出：

---

**参考资料**

[1] Python 官方文档 — 装饰器: https://docs.python.org/3/glossary.html#term-decorator
[2] PEP 318 — Decorators for Functions and Methods: https://peps.python.org/pep-0318/
```

### 9. 整体结构模板

以下是技术博客风格的完整文章结构模板，排版时严格遵循此骨架：

```
# [文章标题]

> **📖 导读**
>
> 本文约 **[字数]** 字，预计阅读 **[分钟]** 分钟。[一句话概括文章价值]。适合 [目标读者] 阅读。

---

## 一、[背景/问题引入]

[1-3 段：为什么要讨论这个话题？当前存在什么问题？建立读者的认知锚点。]

> **💡 核心观点**
>
> [全文最核心的一个观点，让读者一开始就知道这篇文章的「含金量」]

---

## 二、[核心概念/前置知识]

[为后续深入讲解打基础。定义关键术语，澄清常见误解。1-3 段 + 必要时配图。]

---

## 三、[核心内容 — 第一板块]

[正文段落 + 代码 + 说明，段落之间留白充分]

> **💡 核心观点**
>
> [每个核心板块结束时，用一句话提炼要点]

---

## 四、[核心内容 — 第二板块]

[正文段落 + 代码 + 说明]

---

## 五、[延伸思考/实践建议]

[1-3 段：将这些知识应用到实际工作的建议，或对相关方向的展望]

---

## 六、总结

**核心要点回顾：**

- [要点 1]：[一句话归纳第一个核心收获]
- [要点 2]：[一句话归纳第二个核心收获]
- [要点 3]：[一句话归纳第三个核心收获]
- [要点 4]：[一句话归纳第四个核心收获]

**局限与展望**：[1-2 句，诚实说明本文未覆盖的内容或方法论的适用范围]

---

**参考资料**

[1] [资料标题]: [URL]
[2] [资料标题]: [URL]

---

*如果本文对你有帮助，欢迎 **点赞、在看** 支持 👏*
```

## 排版检查清单

- [ ] 文章开头有导读框（📖 导读：字数 + 阅读时间 + 一句话概括）
- [ ] 主章节使用「一、二、三...」中文数字编号
- [ ] 所有代码块标注了语言类型
- [ ] 代码块前后各空一行，且有说明性文字包裹
- [ ] 标题层级正确（## → ###），无跳级
- [ ] 每段不超过 5 行
- [ ] 核心概念首次出现时有粗体强调
- [ ] 全文有 2-4 个「核心观点」框
- [ ] 主要章节间有分割线 `---`
- [ ] 中英文/数字之间有空格
- [ ] 全文 Emoji ≤ 3 个（导读框除外）
- [ ] 有「总结」章节（核心要点 + 局限与展望）
- [ ] 文末有参考资料（脚注式）和互动引导
- [ ] 链接使用脚注式 `[N]` 标记 + 文末集中列出
