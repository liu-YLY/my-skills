# 模板索引与选择指南

> **何时阅读**：阶段 2 匹配风格时必读。
> **覆盖范围**：四种风格的详细描述、适用场景、对比矩阵、推荐建议。
> **可跳过条件**：用户已明确指定风格时跳过，直接读取对应模板文件。

---

## 四种排版风格

### 1. 技术博客风 (`tech-blog`)

**定位**：专业、干净、理性 —— 适合严肃技术内容输出。

**视觉效果**：标题清晰分节、代码块规整、少量点缀色、留白适中。

**适用场景**：
- 编程语言特性解析（如"Python 装饰器深入理解"）
- 技术工具介绍与使用（如"Git 高级技巧"）
- 架构设计与原理讲解
- 技术方案对比

**不适用**：纯经验分享、轻松随笔、无代码片段的纯理论文章。

**详细规则** → [tech-blog.md](tech-blog.md)

---

### 2. 教程指南风 (`tutorial`)

**定位**：清晰、结构化、可操作 —— 适合读者跟着做的内容。

**视觉效果**：步骤编号明确、提示框（TIP / WARNING / NOTE）醒目、代码与说明交替、信息层级强。

**适用场景**：
- 环境搭建 / 配置教程
- 框架入门指南
- 最佳实践操作步骤
- 测试用例编写指南

**不适用**：纯理论分析、经验漫谈、无步骤流程的内容。

**详细规则** → [tutorial.md](tutorial.md)

---

### 3. 深度干货风 (`deep-dive`)

**定位**：深入、严谨、结构强 —— 适合长篇技术分析。

**视觉效果**：多级目录、图表占位、引用规范、关键结论加框、附录与参考。

**适用场景**：
- 源码分析（如"Spring 启动流程源码剖析"）
- 性能优化复盘
- 技术方案设计文档
- 技术攻关经验总结

**不适用**：简短教程、轻松分享、无深度的介绍文章。

**详细规则** → [deep-dive.md](deep-dive.md)

---

### 4. 轻松聊天风 (`casual-chat`)

**定位**：亲和、有趣、易读 —— 适合技术社区分享与个人复盘。

**视觉效果**：短段落、适当 Emoji、口语化表达、对话感强、留白多。

**适用场景**：
- 技术面试经验分享
- 编程学习心得
- 技术行业观察与思考
- Bug 排查趣事

**不适用**：严肃的技术文档、需要严谨引用的内容、企业内部报告。

**详细规则** → [casual-chat.md](casual-chat.md)

---

## 快速选择矩阵

| 你的文章特征 | 推荐风格 |
|-------------|---------|
| 大量代码 + 技术讲解 | `tech-blog` |
| 步骤操作 + 读者需要跟着做 | `tutorial` |
| 深度分析 + 图表 + 引用 | `deep-dive` |
| 经验分享 + 观点 + 轻松氛围 | `casual-chat` |
| 代码 + 步骤操作都有 | `tutorial`（优先） |
| 观点 + 分析 + 少量代码 | `tech-blog` 或 `casual-chat` |

## 选择流程

```
用户文章 → 识别核心内容类型 → 展示 4 种风格简介 → 用户选择 → 加载对应模板
```

**关键规则**：
1. 必须向用户展示四种风格的**一句话简介**，让用户做出选择
2. 可根据文章内容给出**推荐意见**（如"您的文章代码较多、结构清晰，推荐 `tech-blog` 风格"）
3. 不得替用户做最终决定
4. 用户选择后，**立即加载**对应模板文件的完整规则
