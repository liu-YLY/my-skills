# 教程指南风格排版规则

> **何时阅读**：用户选择 `tutorial` 风格时必读。
> **覆盖范围**：步骤化格式、提示框类型、信息层级、整体结构模板。
> **可跳过条件**：未选择此风格时跳过。

---

## 风格定位

清晰、结构化、可操作。让读者能一步步跟着完成操作，每一步都有明确的输入和预期输出。

---

## 排版规则

### 1. 标题规范

- **一级标题** `#`：仅用于文章主标题（对应公众号标题），不在正文中使用
- **二级标题** `##`：用于主要阶段（如 `## 环境准备`、`## 核心步骤`）
- **三级标题** `###`：用于具体步骤（如 `### Step 1: 安装依赖`）
- 三级标题统一以 `Step N:` 或 `第 N 步:` 开头

### 2. 提示框体系（核心特色）

本风格使用 4 种提示框，用引用块 `>` 实现：

```markdown
> **📌 TIP**：提高效率的小技巧。

> **⚠️ WARNING**：可能导致错误或踩坑的地方。

> **💡 NOTE**：补充说明，非必须但推荐了解。

> **✅ CHECK**：完成某步骤后的验证方法。
```

**使用规则**：
- TIP：每 2-3 个步骤配 1 个，提供效率优化建议
- WARNING：每个可能出错的步骤后必须添加
- NOTE：对关键概念做额外解释时使用
- CHECK：每个主要阶段结束时添加验证方法

### 3. 代码块格式

代码块前加说明，代码块后跟预期输出：

```markdown
执行以下命令安装依赖：

```bash
pip install flask sqlalchemy
```

预期输出：

```
Successfully installed flask-2.3.0 sqlalchemy-2.0.0 ...
```

> **✅ CHECK**：执行 `pip list | grep flask` 确认安装成功。
```

### 4. 步骤结构

每个步骤的标准格式：

```markdown
### Step 1: [步骤名称]

**目标**：[这一步要达成什么]

**操作**：

1. [具体操作 1]
2. [具体操作 2]

[代码块（如有）]

**预期结果**：[完成后你应看到什么]

> **⚠️ WARNING**：[常见错误与避免方法]（如有）

> **📌 TIP**：[效率技巧]（如有）
```

### 5. 信息层级

```
## 阶段标题
├── 阶段说明（1-2 句话）
├── ### Step N: 具体步骤
│   ├── 目标
│   ├── 操作
│   ├── 预期结果
│   └── 提示框
└── > ✅ CHECK：阶段验证
```

### 6. 整体结构模板

```
# [教程标题]

## 目标
[2-3 句话说明完成本教程后能实现什么]

## 前置条件
- [需要的基础知识]
- [需要安装的软件/工具]

## 环境准备
...

## 核心步骤
### Step 1: ...
### Step 2: ...
...

## 验证与测试
> ✅ CHECK：[最终验证方法]

## 常见问题 (FAQ)
- **Q: [问题]**
  A: [解答]

## 延伸阅读
- 参考资料1: url
- 参考资料2: url
```

---

## 排版检查清单

- [ ] 标题以 `Step N:` 或 `第 N 步:` 开头
- [ ] 每个易错步骤后有 WARNING 提示框
- [ ] 每个主要阶段结束有 CHECK 验证
- [ ] 代码块后跟预期输出或说明
- [ ] 至少 2 个 TIP 提示框
- [ ] 有 FAQ 章节
- [ ] 「前置条件」已列出
- [ ] 链接使用脚注式或直接标注
